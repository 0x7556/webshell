## 收集各类WebShell服务端

经常看到很多人把服务端和客户端搞混，来个简单科普。

**服务端 vs. 控制端**

- 服务端WebShell：像餐厅里的服务员，负责接收请求并给出响应。它提供功能、处理数据并返回结果。
- 控制端（客户端）：像点餐的顾客，发出请求或下达指令，等待服务端响应并展示或使用结果。

远控RAT\C2同理，无论国内还是国外，有很多人把C2客户端写成服务端，人类在操作打命令的是客户端。服务端是种在别人电脑上的马，它一直在等待人类下达指令，就是服务端，很好理解啊，为什么很多人都搞错呢？
